## D001 — Estrutura modular

**Data:** 08/07/2026

### Decisão

Organizar o projeto utilizando uma estrutura modular, separando cada responsabilidade em um módulo específico.

### Motivo

A separação de responsabilidades facilita manutenção, testes, reutilização de código e futuras expansões do sistema.

### Consequências

- Código mais organizado.
- Melhor escalabilidade.
- Facilidade para escrever testes unitários.
